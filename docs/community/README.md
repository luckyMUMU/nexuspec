## 贡献指南

我们欢迎各种形式的贡献！

### 报告 Bug

如果你发现了 Bug，请按以下步骤报告：

1. 首先在 [Issues](https://github.com/nexus-spec/cli/issues) 中搜索，确认没有重复
2. 使用 Bug 报告模板创建新 Issue
3. 包含尽可能多的信息：
   - NexusSpec 版本
   - Node.js 版本
   - 操作系统
   - 复现步骤
   - 预期行为
   - 实际行为
   - 错误信息（如果有）

**Bug 报告模板：**

```markdown
## 描述
清晰简洁地描述这个 Bug。

## 复现步骤
1. ...
2. ...
3. ...

## 预期行为
你期望发生什么？

## 实际行为
实际发生了什么？

## 环境
- NexusSpec 版本：
- Node.js 版本：
- 操作系统：

## 其他信息
任何其他相关信息、截图等。
```

### 提交功能请求

如果你有新功能的想法：

1. 在 [Issues](https://github.com/nexus-spec/cli/issues) 中搜索，确认没有类似的请求
2. 使用功能请求模板创建新 Issue
3. 包含以下信息：
   - 功能描述
   - 解决的问题
   - 建议的解决方案
   - 备选方案（如果有）

**功能请求模板：**

```markdown
## 功能描述
清晰简洁地描述你想要的功能。

## 解决的问题
这个功能解决了什么问题？

## 建议的方案
你希望如何实现？

## 备选方案
你考虑过其他方案吗？

## 其他信息
任何其他相关信息。
```

### 贡献代码

如果你想贡献代码：

1. Fork 仓库
2. 克隆你的 Fork
3. 创建功能分支
4. 提交更改
5. 推送到你的 Fork
6. 创建 Pull Request

**开发设置：**

```bash
# 克隆仓库
git clone https://github.com/nexus-spec/cli.git
cd cli

# 安装依赖
npm install

# 构建
npm run build

# 测试
npm test

# 运行开发服务器
npm run dev
```

**PR 指南：**
- 每个 PR 应该专注于一个功能或修复
- 写清楚的提交信息
- 包含相关测试（如果适用）
- 更新文档（如果需要）
- 确保所有测试通过

**代码风格：**
项目使用 ESLint 和 Prettier，确保你的代码符合这些规范。

## 常见问题

### 安装和设置

**Q: NexusSpec 支持哪些 Node.js 版本？**
A: NexusSpec 要求 Node.js 20.19.0 或更高版本。

**Q: 我可以在 Windows 上使用 NexusSpec 吗？**
A: 可以！NexusSpec 支持 Windows、macOS 和 Linux。

**Q: 如何更新 NexusSpec？**
A: 运行 `npm update -g @nexus-spec/cli`。

### 命名空间

**Q: 命名空间仓库必须托管在 GitHub 吗？**
A: 不是！你可以使用任何 Git 托管服务，包括 GitLab、Bitbucket，甚至是本地仓库。

**Q: 我可以在多个命名空间中使用同一个服务吗？**
A: 目前每个服务只能属于一个命名空间。如有需要，可以考虑合并命名空间。

### 契约管理

**Q: 契约版本号是如何确定的？**
A: NexusSpec 使用简单的递增版本号（v1, v2, v3...），而不是语义化版本号。

**Q: 我可以直接修改已发布的契约吗？**
A: 不建议！应该通过新版本来修改，并按照提案流程进行。

**Q: 如何知道哪些服务在使用旧版本的契约？**
A: 使用 `nxsp contract outdated <service> <type> <name>` 命令。

### 提案流程

**Q: 提案的审查时间通常是多久？**
A: 这取决于团队的响应速度，建议在创建提案前先进行沟通。

**Q: 可以拒绝提案吗？**
A: 可以！目标服务可以选择接受或拒绝提案。如果拒绝，应该提供清晰的理由。

**Q: 提案被拒绝后怎么办？**
A: 可以根据反馈修改提案后重新提交，或者寻找其他解决方案。

### CI/CD 集成

**Q: NexusSpec 支持哪些 CI/CD 平台？**
A: NexusSpec 提供 GitHub Actions 配置，也可以与其他 CI/CD 平台集成。

**Q: 如何在 CI/CD 中验证契约兼容性？**
A: 使用 `nxsp cicd validate` 命令。

### 故障排除

**Q: 遇到 "cannot find namespace" 错误怎么办？**
A: 检查 `.nxsp/config.yaml` 中的 `namespace.remote` 配置是否正确。

**Q: 同步契约时出现 Git 错误怎么办？**
A: 确保你有命名空间仓库的写入权限，并且本地分支是最新的。

**Q: 如何调试 NexusSpec？**
A: 可以设置环境变量 `NXSP_DEBUG=true` 来启用调试日志。
