# 使用指南

## 如何使用Web-Dev技能

### 1. 准备工作

- 确保工作区为空或不包含前端代码
- 明确项目需求和目标
- 准备好项目的基本信息，如项目名称、功能需求等

### 2. 调用Web-Dev技能

当您需要创建全新的Web项目时，调用Web-Dev技能。技能会自动执行以下步骤：

#### 步骤1：检查现有文档

技能会首先检查`.trae/documents/`目录中是否存在PRD和技术架构文档：
- 如果两个文档都存在：直接跳转到开发阶段
- 如果文档缺失或不完整：继续创建必要的文档

#### 步骤2：创建必要文档

如果文档缺失，技能会：
1. 读取并遵循`web-docs-guideline.md`中的约束
2. 生成产品需求文档（PRD）
3. 生成技术架构文档
4. 将所有文档存储在`.trae/documents/`目录中

#### 步骤3：等待用户批准

文档生成后，技能会通知用户文档已准备就绪，需要用户批准后才能进入开发阶段。

#### 步骤4：开发阶段

用户批准后，技能会：
1. 读取`web-dev-guideline.md`并严格遵循其中的指南
2. 根据PRD和技术架构实现解决方案
3. 应用设计指南，创建视觉吸引力强的界面
4. 遵循所有技术约束和架构决策

## 输入格式

当调用Web-Dev技能时，您可以提供以下信息：

- **项目描述**：简要描述项目的目的、功能和目标用户
- **技术要求**：如果有特定的技术栈要求，明确说明
- **设计风格**：如果有特定的设计风格偏好，提供相关信息
- **功能需求**：详细列出项目需要实现的功能

## 最佳实践

### 文档创建最佳实践

1. **清晰的项目描述**：提供详细的项目描述，包括目标用户、核心功能和业务价值
2. **详细的功能需求**：列出所有必要的功能，包括优先级
3. **合理的技术选择**：根据项目需求选择合适的技术栈
4. **设计风格明确**：提供具体的设计风格要求，包括色彩、排版等

### 开发最佳实践

1. **遵循文档**：严格按照PRD和技术架构文档进行开发
2. **代码质量**：保持代码整洁、可维护，添加适当的注释
3. **响应式设计**：确保界面在不同设备上都能正常显示
4. **性能优化**：优化代码性能，减少加载时间
5. **用户体验**：关注用户体验，确保界面直观、易用

### 设计最佳实践

1. **明确的美学方向**：选择一个清晰的设计风格，并在整个项目中保持一致
2. **排版选择**：选择美观、独特的字体，避免通用字体
3. **色彩方案**：使用一致的色彩方案，确保色彩搭配协调
4. **动效设计**：添加适当的动画效果，提升用户体验
5. **空间利用**：合理利用空间，创建视觉层次感

## 常见问题

### Q: Web-Dev技能适合什么样的项目？
A: Web-Dev技能适合从0到1的绿色场域Web开发项目，如创建全新的网站、Web应用或Web游戏。

### Q: 我可以使用Web-Dev技能修改现有项目吗？
A: 不建议使用Web-Dev技能修改现有项目。对于现有项目的修改，建议使用常规开发工作流程。

### Q: Web-Dev技能支持哪些技术栈？
A: Web-Dev技能默认推荐React@18 + Tailwind CSS@3 + Vite，也支持其他前端框架和工具。

### Q: 文档生成后，我可以修改吗？
A: 是的，您可以根据需要修改生成的文档，然后再次调用技能进行开发。

### Q: Web-Dev技能支持3D效果吗？
A: 是的，对于需要3D效果的项目，Web-Dev技能提供Three.js等库的集成支持。
