# 入门教程：Web-Dev技能使用指南

## 简介

本教程旨在帮助初学者快速上手Web-Dev技能，了解其基本功能和使用方法。通过本教程，您将学习如何使用Web-Dev技能创建全新的Web项目。

## 前提条件

- 基本的Web开发知识（HTML、CSS、JavaScript）
- 空的工作区或不包含前端代码的工作区
- 明确的项目需求

## 步骤1：了解Web-Dev技能

### 什么是Web-Dev技能？

Web-Dev技能是一个专门用于创建生产级Web界面的工具，具有高质量的设计能力。它专为从0到1的绿色场域Web开发场景设计，帮助用户快速创建全新的Web项目、网站、网页、Web应用或Web游戏。

### 适用场景

- 创建全新的Web项目
- 工作区为空或不包含前端代码
- 希望遵循完整的Web开发工作流程

### 不适用场景

- 修改现有前端项目
- 对现有代码进行bug修复
- 增量开发已建立的代码库

## 步骤2：调用Web-Dev技能

### 输入格式

当调用Web-Dev技能时，您需要提供以下信息：

1. **项目描述**：简要描述项目的目的、功能和目标用户
2. **技术要求**：如果有特定的技术栈要求，明确说明
3. **设计风格**：如果有特定的设计风格偏好，提供相关信息
4. **功能需求**：详细列出项目需要实现的功能

### 示例输入

```
我需要创建一个个人作品集网站，展示我的项目和技能。网站应该包含首页、项目页面和关于我页面。设计风格要现代、简洁，使用React和Tailwind CSS。
```

## 步骤3：文档生成

### 检查现有文档

Web-Dev技能会首先检查`.trae/documents/`目录中是否存在PRD和技术架构文档：
- 如果两个文档都存在：直接跳转到开发阶段
- 如果文档缺失或不完整：继续创建必要的文档

### 生成PRD

如果文档缺失，Web-Dev技能会生成产品需求文档（PRD），包含：
- 产品概述
- 核心功能
- 核心流程
- 用户界面设计

### 生成技术架构文档

技术架构文档包含：
- 架构设计
- 技术描述
- 路由定义
- API定义（如适用）
- 服务器架构图（如适用）
- 数据模型（如适用）

### 等待用户批准

文档生成后，Web-Dev技能会通知您文档已准备就绪，需要您批准后才能进入开发阶段。

## 步骤4：开发阶段

### 查看文档

在开始开发前，建议您查看生成的文档，确保它们符合您的需求。如果需要修改，可以直接编辑文档。

### 开发过程

Web-Dev技能会根据PRD和技术架构文档实现解决方案：
1. 初始化项目
2. 安装必要的依赖
3. 实现页面和功能
4. 应用设计风格
5. 测试和优化

### 开发完成

开发完成后，您将获得一个完整的Web项目，包括：
- 功能完整的前端代码
- 响应式设计
- 现代的UI/UX
- 性能优化

## 步骤5：项目部署

### 构建项目

使用以下命令构建项目：

```bash
npm run build
```

### 部署选项

- **静态网站托管**：如Vercel、Netlify、GitHub Pages
- **云服务**：如AWS、Google Cloud、Azure
- **容器化**：使用Docker部署

## 常见问题

### Q: 文档生成后，我可以修改吗？
A: 是的，您可以根据需要修改生成的文档，然后再次调用技能进行开发。

### Q: Web-Dev技能支持哪些技术栈？
A: Web-Dev技能默认推荐React@18 + Tailwind CSS@3 + Vite，也支持其他前端框架和工具。

### Q: 如何指定设计风格？
A: 在调用技能时，您可以提供具体的设计风格要求，如现代、简洁、复古等。

### Q: 项目开发完成后，如何进行后续维护？
A: 您可以使用常规的前端开发工作流程进行后续维护和更新。
